# Release Checklist Template

## Pre-Release

- Tests pass.
- Security audits pass or accepted exceptions are documented.
- API specs are updated.
- Atlas context is updated.

## Release

- Submodule commits pushed.
- Root submodule pointers pushed.
- Deployment sync scripts run only if deployment is intended.

## Post-Release

- Actions checked.
- Deployment checked.
- Known issues recorded.
